# Agent Harness 综述报告

## 1. 背景概览

Agent Harness 是一种基础设施，用于长期稳定地运行 AI 模型。它结合了模型与操作系统，增强代理的能力并管理复杂任务。在 AI 工程化中，Agent Harness 起着至关重要的作用，它不仅提高了模型的稳定性和可靠性，还降低了 AI 工程化门槛。

## 2. 核心洞见

1. **定义与核心概念**：Agent Harness 是一种基础设施，结合模型与操作系统，增强代理能力并管理复杂任务。[来源：Agent Harness到底是什么 - 知乎专栏](https://zhuanlan.zhihu.com/p/2011508456717304293)
2. **Harness 的功能模块**：包括规划与编排引擎、沙箱执行环境、技能与工具系统、记忆与上下文工程、系统提示词与角色准则、可观测性与反馈闭环。[来源：54k+Star 爆火！ AI “新王者框架 “Harness Agent 来了！尼恩来一次Harness穿透式解读 - 技术自由圈 - 博客园](https://www.cnblogs.com/crazymakercircle/p/19856262)
3. **Harness 与 Agent 的关系**：Harness 是操作系统，负责整理上下文、处理“启动”序列，并提供标准驱动；Agent 是应用程序，运行在操作系统之上的具体用户逻辑。[来源：一文读懂Harness Engineering：从14篇工程文章探寻让AI循规蹈矩的 ...](https://eu.36kr.com/zh/p/3749464991187458)
4. **Harness 的优势**：提高模型的稳定性和可靠性，降低 AI 工程化门槛，改变工程师的工作重心。[来源：LLM - 一文读懂Agent Harness-CSDN博客](https://blog.csdn.net/yangshangwei/article/details/158586125)
5. **Harness 的应用案例**：Claude Code、DeerFlow 2.0、Claude Agent SDK。[来源：Harness Engineering 深度解析：AI Agent 时代的工程范式革命- 知乎](https://zhuanlan.zhihu.com/p/2014014859164026634)

## 3. 证据与数据

- Agent Harness 的核心功能包括管理、调度和监控 AI 代理，使其能够作为企业生产力工具运行。
- Harness 由 Guides（前馈控制）和 Sensors（反馈控制）两部分组成。
- Agent Harness 在多个领域得到应用，包括编程、内容创作、数据分析、客户服务和企业自动化。
- Agent Harness 的开源项目如 oh-my-claudecode、oh-my-openagent、oh-my-codex 和 oh-my-pi 等展示了其在不同 Agent 引擎上的应用和创新。
- Anthropic 推出的 Claude Managed Agents 是一个托管式 harness 服务，提供了预构建、可配置的 Agent Harness。

## 4. 风险与挑战

- Agent Harness 的设计和实现可能非常复杂，需要专业的技术知识和经验。
- 安全性和隔离性是 Agent Harness 的核心挑战，需要确保 AI 模型在执行任务时的可靠性和安全性。
- Harness Engineering 的竞争激烈，需要不断创新和改进以保持竞争力。
