## 报告漏洞

请 **不要** 在公开 Issue 中提交安全问题。

请通过以下任一方式私下联系维护者:
- 发送邮件到仓库主页列出的地址
- 在 GitHub 上提交私有安全公告: *Security → Advisories → New draft security advisory*

报告中请包含:
- 简要的影响描述
- 复现步骤或概念验证 (PoC)
- 受影响的版本

我们承诺在 3 个工作日内回复新报告,关键问题将在 30 天内发布修复。

## 密钥管理

应用通过环境变量读取 API 密钥(`TAVILY_API_KEY`、`PERPLEXITY_API_KEY`、`LLM_API_KEY` 等)。
**永远不要** 提交 `.env` 文件 —— `.env.example` 是唯一应该入库的配置文件。
